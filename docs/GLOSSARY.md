# 용어 사전 (Glossary)

## A

### API (Application Programming Interface)
소프트웨어 간 통신을 위한 인터페이스. 본 시스템에서는 REST API 사용.

### Assertion
테스트에서 예상 결과가 실제 결과와 일치하는지 확인하는 문장.

---

## B

### BIS (Bus Information System)
정류장 종합 관리 및 실시간 모니터링 시스템.

### Backend
서버 측 응용 프로그램. API, 데이터베이스 등을 포함.

---

## C

### CMS (Content Management System)
정류장 단말에 표시될 콘텐츠를 관리하는 시스템.

### Component
React의 재사용 가능한 UI 단위. 버튼, 카드, 폼 등.

---

## D

### Dashboard
시스템 전체 현황을 한눈에 보여주는 화면.

### Dialog / Drawer
사용자 입력을 받거나 상세 정보를 표시하는 팝업 윈도우.

### DevOps
개발(Dev)과 운영(Ops)을 통합하여 효율성을 높이는 문화/프로세스.

---

## E

### E2E Test (End-to-End Test)
사용자 관점에서 전체 애플리케이션 흐름을 테스트.

### Entity
데이터베이스의 테이블, 또는 비즈니스 도메인의 객체. 예) 자산, 단말, 사용자.

---

## F

### Frontend
사용자가 직접 상호작용하는 클라이언트 측 애플리케이션. Portal, Tablet 포함.

### Field-Ops (Field Operations)
현장 설치, 유지보수 등 현장 작업을 관리하는 모듈.

---

## G

### GPS (Global Positioning System)
위치 정보 시스템. 정류장 위치 파악에 사용.

### Geolocation
기기의 현재 위치를 획득하는 기능.

---

## H

### HTTP / HTTPS
인터넷 통신 프로토콜. HTTPS는 암호화된 보안 통신.

---

## I

### Integration Test
여러 모듈의 상호작용을 테스트.

### Issue
버그, 기능 요청 등 해결해야 할 작업.

---

## J

### JWT (JSON Web Token)
사용자 인증을 위한 토큰 방식.

---

## K

### KPI (Key Performance Indicator)
핵심 성과 지표. 대시보드의 "오늘 작업 수", "긴급 출동 수" 등.

---

## L

### Locale / Localization
언어/지역 설정. 다국어 지원에 필요.

### Logging
애플리케이션 실행 정보를 기록하는 것.

---

## M

### Middleware
요청/응답 사이에서 처리되는 중간 소프트웨어.

### Mock Data
실제 데이터 대신 테스트를 위해 임의로 만든 데이터.

### Module
특정 기능을 담당하는 독립적인 소프트웨어 단위. Registry, Field-Ops 등.

---

## N

### Navigation
화면 간 이동. Tablet의 지도 앱 네비게이션과는 다른 개념.

---

## O

### ORM (Object-Relational Mapping)
데이터베이스 테이블을 객체로 매핑하여 SQL을 추상화.

### Outbox Pattern
오프라인 동기화를 위해 변경 사항을 임시 저장소에 모았다가 온라인 시 전송.

---

## P

### Portal
관리자/운영자가 사용하는 웹 기반 종합 관리 시스템.

### Provider
데이터 제공자. RMS Provider, CMS Provider 등.

---

## Q

### Query String
URL에 포함된 파라미터. ?id=1&name=test

---

## R

### RMS (Realtime Management System)
단말의 실시간 모니터링 및 원격 제어 시스템.

### Registry
자산, 단말, 고객사, 정류장 등 마스터 데이터를 관리하는 모듈.

### REST API
Representational State Transfer. HTTP 메서드(GET, POST, PUT, DELETE)를 이용한 API.

### Rollback
배포 후 문제 발생 시 이전 버전으로 되돌리는 것.

---

## S

### Schema
데이터베이스 테이블 구조 정의.

### Sprint
Agile 개발의 반복 주기. 보통 2주.

### State Management
애플리케이션의 상태(데이터)를 관리하는 패턴.

### SWR (Stale-While-Revalidate)
데이터 페칭 라이브러리. 캐시된 데이터를 먼저 표시하고 백그라운드에서 갱신.

---

## T

### Tablet
현장 기술자가 사용하는 모바일 애플리케이션.

### Type / TypeScript
프로그래밍 언어. 정적 타입 검사로 버그 방지.

---

## U

### Unit Test
함수/컴포넌트 단위의 가장 작은 테스트.

### UI / UX
UI: User Interface (사용자 인터페이스)
UX: User Experience (사용자 경험)

---

## V

### ViewModel
비즈니스 로직과 UI를 분리하기 위한 패턴.

### Version Control
Git 등 소스 코드 버전을 관리하는 시스템.

---

## W

### Webhook
외부 이벤트 발생 시 자동으로 호출되는 HTTP 콜백.

### Workflow
작업 흐름. 사용자의 작업 단계를 정의.

---

## X

### XML / XSS
XML: 마크업 언어
XSS: Cross-Site Scripting 공격

---

## Z

### Zero-Trust Security
모든 요청을 신뢰하지 않고 검증하는 보안 모델.
