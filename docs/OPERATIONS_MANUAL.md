# 운영 매뉴얼 (Operations Manual)

## 1. 일일 운영 체크리스트

### 오전
- [ ] 시스템 상태 확인 (대시보드)
- [ ] 긴급 장애 확인
- [ ] 오늘 할당된 작업 검토
- [ ] 날씨/외부 요인 확인 (필요시)

### 오후
- [ ] 진행 중인 작업 상태 확인
- [ ] 완료된 작업 검증
- [ ] 내일 일정 준비

### 종료 전
- [ ] 데이터 백업 확인
- [ ] 에러 로그 검토
- [ ] 알려진 이슈 추적

---

## 2. 주간 운영

### 월요일
- 주간 계획 수립
- 자산 현황 점검
- 인력 배치 계획

### 수요일
- 중간 진행 상태 확인
- 필요시 일정 조정

### 금요일
- 주간 성과 평가
- 다음 주 계획

---

## 3. 월간 운영

### 월초 (1~3일)
- 지난 달 통계 분석
- 이번 달 목표 설정
- 리소스 재배치

### 월중 (10~15일)
- 중간 실적 검토
- 문제점 분석
- 개선 방안 수립

### 월말 (25~31일)
- 월간 보고서 작성
- 데이터 아카이빙
- 다음 달 준비

---

## 4. 사용자 관리

### 새 사용자 온보딩
1. 계정 생성
2. 권한 설정
3. 담당 지역/정류장 배정
4. 교육 자료 제공
5. 시스템 접근 권한 활성화

### 사용자 제거
1. 권한 회수
2. 할당 작업 재배치
3. 계정 비활성화

---

## 5. 장애 대응

### 심각도 분류
| 수준 | 영향 | 대응 시간 |
|------|------|---------|
| 긴급 | 시스템 전체 중단 | 15분 |
| 높음 | 주요 기능 불가 | 1시간 |
| 중간 | 부분 기능 불가 | 4시간 |
| 낮음 | 경미한 결함 | 24시간 |

### 대응 절차
1. 장애 감지
2. 영향도 평가
3. 초기 대응 (재시작, 캐시 초기화)
4. 상황실 보고
5. 해결
6. 원인 분석 및 보고

---

## 6. 백업 및 복구

### 백업 정책
- **일일**: 매일 자동 백업
- **주간**: 주 1회 수동 검증
- **월간**: 월 1회 전체 아카이빙

### 복구 절차
1. 백업 포인트 선택
2. 테스트 환경에서 복구 검증
3. 프로덕션 복구
4. 데이터 정합성 확인

---

## 7. 성능 모니터링

### 주요 지표
- **응답 시간**: 목표 < 500ms
- **가용성**: 목표 99.9%
- **에러율**: 목표 < 0.1%

### 모니터링 도구
- Vercel Analytics
- Google Analytics
- Sentry (에러 추적)

---

## 8. 보안 관리

### 정기 점검
- 주 1회: 로그 검토
- 월 1회: 권한 감사
- 분기 1회: 보안 테스트

### 인증서 관리
- SSL 인증서 만료일 주시
- 자동 갱신 설정 확인

---

## 9. 성능 최적화

### 정기 작업
- 데이터베이스 인덱스 재구성
- 불필요한 데이터 정리
- 캐시 최적화

### 분석
- 월 1회: 느린 쿼리 분석
- 분기 1회: 번들 크기 검토

---

## 10. 관계자 보고

### 주간 보고
- 완료 작업 건수

### 분기 보고
