# 장애 대응 가이드 (Troubleshooting Guide)

## 1. 시스템 가동 중단

### 증상
- Portal 또는 Tablet 접속 불가
- "연결할 수 없습니다" 오류

### 대응
1. **상태 확인**
   ```bash
   curl https://yourdomain.com/api/health
   ```

2. **Vercel 상태 확인**
   - https://www.vercel.com/status

3. **초기 조치**
   - 브라우저 캐시 삭제
   - 다른 네트워크에서 접속 시도

4. **이슈 보고**
   - Sentry 로그 확인
   - Vercel 대시보드 배포 로그 확인

---

## 2. 느린 응답

### 증상
- 페이지 로딩 5초 이상 소요
- API 응답 지연

### 대응
1. **네트워크 확인**
   ```bash
   ping yourdomain.com
   ```

2. **데이터베이스 확인**
   - 느린 쿼리 로그 검토
   - 인덱스 상태 확인

3. **캐시 확인**
   - 캐시 히트율 검토
   - 캐시 무효화 시간 검토

4. **번들 크기 확인**
   ```bash
   npm run analyze
   ```

---

## 3. API 오류

### 404 Not Found
```
원인: 엔드포인트 없음 또는 리소스 삭제
대응: URL 확인, API 명세서 검토
```

### 500 Internal Server Error
```
원인: 서버 에러
대응: 서버 로그 확인, Sentry 검토
```

### 401 Unauthorized
```
원인: 인증 실패
대응: 로그인 다시 시도, 토큰 갱신
```

### 403 Forbidden
```
원인: 권한 부족
대응: 사용자 권한 확인, 관리자에게 보고
```

---

## 4. 데이터베이스 연결 오류

### 증상
- "데이터베이스 연결 실패"
- 데이터 조회 불가

### 대응
1. **연결 문자열 확인**
   ```bash
   echo $DATABASE_URL
   ```

2. **데이터베이스 상태**
   ```sql
   SELECT 1;  -- 연결 테스트
   ```

3. **연결 풀 확인**
   - 활성 연결 수 확인
   - 타임아웃 설정 검토

4. **방화벽 확인**
   - IP 화이트리스트 확인
   - 포트 개방 확인

---

## 5. 인증 문제

### 로그인 실패
```
1. 이메일 주소 확인
2. 비밀번호 초기화
3. 계정 활성화 상태 확인
4. 브라우저 쿠키 삭제
```

### 토큰 만료
```
1. 페이지 새로고침
2. 다시 로그인
3. 토큰 갱신 엔드포인트 확인
```

### 권한 오류
```
1. 사용자 역할 확인
2. 권한 범위 확인
3. 관리자에게 권한 요청
```

---

## 6. Tablet 앱 문제

### 앱이 켜지지 않음
```
1. 앱 캐시 삭제
2. 앱 재설치
3. OS 업데이트 확인
4. 기기 재부팅
```

### 지도가 안 보임
```
1. GPS 권한 확인
2. 위치 서비스 활성화
3. Google Maps API 키 확인
4. 인터넷 연결 확인
```

### 점검 기록 업로드 실패
```
1. 네트워크 연결 확인
2. Outbox 동기화 시도
3. 용량 확인 (사진 크기)
4. 시스템 동기화 대기
```

---

## 7. 지도 관련 문제

### 지도 로딩 안 됨
```
원인: API 키 만료 또는 할당량 초과
대응:
1. Google Cloud Console 확인
2. API 할당량 확인
3. 결제 정보 확인
4. API 키 재발급
```

### 마커 표시 안 됨
```
원인: GPS 좌표 오류 또는 API 문제
대응:
1. 좌표값 검증
2. Google Maps API 호출 확인
3. 브라우저 콘솔 에러 확인
```

## 8. 성능 문제

### 메모리 누수
```bash
# Chrome DevTools
Performance > Memory > Take heap snapshot
```

### CPU 높음
```
1. 불필요한 렌더링 확인
2. 이벤트 리스너 제거 확인
3. 무한 루프 확인
```

---

## 9. 보안 문제

### CORS 오류
```
원인: 크로스 도메인 요청 차단
대응:
1. CORS 설정 확인
2. API 서버 CORS 헤더 추가
```

### XSS 공격 의심
```
대응:
1. 입력값 검증
2. 출력값 인코딩
3. CSP 헤더 설정
```

### SQL Injection 의심
```
대응:
1. Parameterized Query 사용 확인
2. 입력값 검증
3. 데이터 감시 활성화
```

---

## 10. 배포 문제

### 배포 실패
```
1. 빌드 로그 확인
2. 환경 변수 확인
3. 의존성 버전 확인
4. 이전 버전으로 롤백
```

### 배포 후 오류
```
1. Sentry 에러 확인
2. 최근 변경사항 검토
3. 긴급 롤백 시행
```

---

## 11. 모니터링 및 알림

### 알림 설정
- Sentry: 에러 알림
- Vercel: 배포/성능 알림
- 커스텀: 백그라운드 작업 모니터링

### 대시보드 확인 주기
- 실시간: 긴급 장애
- 매시간: 성능 지표
- 일일: 에러 로그
- 주간: 전체 검토

---

## 12. 에스컬레이션

### Level 1 (개발팀)
- 응답 시간: 15분
- 권한: 임시 해결, 로그 분석

### Level 2 (시스템 관리자)
- 응답 시간: 1시간
- 권한: 데이터베이스 관리, 인프라 조정

### Level 3 (CTO)
- 응답 시간: 30분
- 권한: 배포 결정, 롤백 결정
