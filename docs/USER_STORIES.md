# 사용자 스토리 (User Stories)

## 1. Portal - 운영자

### US-001: 작업 배정
**역할**: 운영자
**요구**: 새로운 설치 작업을 기술자에게 배정하고 싶다.

**수락 조건**:
- 작업 유형, 위치, 예정 날짜 입력 가능
- 기술자 선택 가능
- 작업 저장 시 Tablet에 자동 동기화

---

### US-002: 단말 모니터링
**역할**: 운영자
**요구**: 모든 단말의 실시간 상태를 한눈에 보고 싶다.

**수락 조건**:
- 정상/경고/오류 단말 수 표시
- 지도에 단말 위치 표시
- 장애 단말 클릭 시 상세 정보 조회

---

### US-003: 장애 대응
**역할**: 운영자
**요구**: 장애 정보를 우선순위별로 보고 빠르게 대응하고 싶다.

**수락 조건**:
- 우선순위별 정렬 (긴급/높음/중간/낮음)
- 원격 제어 명령어 전송 가능
- 대응 이력 기록

---

## 2. Portal - 관리자

### US-004: 사용자 권한 설정
**역할**: 관리자
**요구**: 각 사용자에게 역할과 권한을 할당하고 싶다.

**수락 조건**:
- 사용자 생성/수정 가능
- 역할 선택 (관리자/운영자/기술자/조회자)
- 데이터 접근 범위 제한

---

### US-005: 감사 로그 조회
**역할**: 관리자
**요구**: 언제, 누가, 무엇을 했는지 감시하고 싶다.

**수락 조건**:
- 사용자별 로그 조회
- 기간별 필터
- 로그 내보내기

---

## 3. Tablet - 기술자

### US-006: 오늘 할 일 확인
**역할**: 기술자
**요구**: 출근 후 오늘 할 일을 빠르게 확인하고 싶다.

**수락 조건**:
- 오늘 작업 목록 표시
- 각 작업의 위치, 시간, 내용 표시
- 지도에서 위치 확인

---

### US-007: 현장에서 점검 기록
**역할**: 기술자
**요구**: 정류장 점검 후 기록을 앱에서 바로 등록하고 싶다.

**수락 조건**:
- 점검 유형 선택
- 점검 결과 입력
- 현장 사진 첨부
- 기록 저장 (오프라인 지원)

---

### US-008: 길찾기
**역할**: 기술자
**요구**: 정류장으로 가는 길을 네비게이션하고 싶다.

**수락 조건**:
- "길찾기" 버튼 클릭
- 지도 앱 선택 (카카오맵/네이버/구글)
- 네비게이션 시작

---

### US-009: 작업 완료 보고
**역할**: 기술자
**요구**: 현장에서 작업 완료 후 완료 정보를 입력하고 싶다.

**수락 조건**:
- 완료 시간 자동 기록
- 점검 결과 입력
- 현장 사진 첨부
- 상태 변경 (진행중 → 완료)

---

### US-010: 재고 출고
**역할**: 기술자 (설치팀)
**요구**: 설치하려는 부품을 창고에서 받을 때 출고 기록을 남기고 싶다.

**수락 조건**:
- 부품명, 수량 입력
- 출고 기록 저장
- 창고팀이 시스템에서 확인 가능

---

## 4. Tablet - 창고팀

### US-011: 입고 기록
**역할**: 창고팀
**요구**: 새로운 부품이 도착했을 때 입고를 기록하고 싶다.

**수락 조건**:
- 부품명, 수량, 공급업체 입력
- 입고 기록 저장
- 재고량 자동 갱신

---

### US-012: 재고 현황 조회
**역할**: 창고팀
**요구**: 창고에 있는 부품의 현재 재고를 확인하고 싶다.

**수락 조건**:
- 부품별 재고 수량 표시
- 부족한 부품 표시
- 입출고 이력 조회

---

## 5. Tablet - CMS 담당자

### US-013: 정류장 콘텐츠 관리
**역할**: Portal 운영자 (CMS)
**요구**: 정류장에 표시될 공지사항을 관리하고 싶다.

**수락 조건**:
- 콘텐츠 작성/수정
- 특정 정류장 선택
- 게시 일시 설정
- 즉시/예약 게시
