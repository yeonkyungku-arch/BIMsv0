# 단말 등록 - 정류장 연동 - 설치 구축 프로세스

> 대상: 개발자 / 운영자  
> 버전: 1.0  
> 최종 수정: 2026-03-24

---

## 1. 프로세스 개요

BIS 단말의 등록부터 정상 운영까지의 전체 라이프사이클을 정의합니다.

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. 사전    │ → │  2. 단말    │ → │  3. 설치    │ → │  4. 그룹    │
│     준비    │    │  등록/연동  │    │     작업    │    │  배정/운영  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 2. 상세 프로세스

### 2.1 사전 준비 (Registry 마스터 데이터)

| 순서 | 메뉴 | 경로 | 설명 |
|------|------|------|------|
| 1-1 | 파트너 관리 | `/registry/partners` | 설치/유지보수 업체 등록 |
| 1-2 | 고객사 관리 | `/registry/customers` | 교통공사, 지자체 등록 |
| 1-3 | 정류장 관리 | `/registry/stops` | 정류장 위치, BIS API ID 등록 |

**선행 조건**: 없음  
**완료 조건**: 파트너, 고객사, 정류장 기본 정보 등록 완료

---

### 2.2 단말 등록 및 연동

| 순서 | 작업 | 경로 | 상태 변화 | 설명 |
|------|------|------|----------|------|
| 2-1 | 단말 정보 입력 | `/registry/devices` | → `REGISTERED` | 시리얼, 전원유형(태양광/전력형) 입력 |
| 2-2 | 정류장 연동 | `/registry/relationships` | → `LINKED` | 단말 ↔ 정류장 매핑 |

**선행 조건**: 정류장 등록 완료  
**완료 조건**: 단말-정류장 연동 완료

---

### 2.3 설치 작업 (Field Operations)

| 순서 | 작업 | 경로 | 상태 변화 | 담당 |
|------|------|------|----------|------|
| 3-1 | 작업지시서 생성 | `/field-ops/work-orders` | → `CREATED` | 관리자 |
| 3-2 | 기사 배정 | `/field-ops/work-orders` | → `ASSIGNED` | 관리자 |
| 3-3 | 현장 도착/작업 시작 | `/tablet/work-orders` | → `IN_PROGRESS` | 현장 기사 |
| 3-4 | 설치 완료 보고 | `/tablet/work-orders` | → `COMPLETION_SUBMITTED` | 현장 기사 |
| 3-5 | 검수 승인 | `/field-ops/installations` | → `VERIFIED` | 관리자 |

**선행 조건**: 단말-정류장 연동 완료  
**완료 조건**: 설치 검수 승인, 정상 작동 확인

### 2.4 그룹 배정 및 운영

| 순서 | 작업 | 경로 | 상태 변화 | 설명 |
|------|------|------|----------|------|
| 4-1 | BIS 그룹 생성 | `/registry/groups` | - | 운영 단위 정의 |
| 4-2 | 검증 단말 배정 | `/registry/groups` | → `OPERATING` | 정상 작동 확인된 단말만 배정 |
| 4-3 | 정상 운영 | - | - | CMS 배포, RMS 모니터링 대상 |

**선행 조건**: 설치 검수 승인 완료 (VERIFIED 상태)  
**완료 조건**: 그룹 배정 완료, 운영 시작

## 3. 단말 상태 전이도

```
                                    ┌─────────────────────────┐
                                    │    INSTALLATION_FAILED  │
                                    │      (설치 실패)         │
                                    └───────────▲─────────────┘
                                                │
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──┴───────┐   ┌──────────┐   ┌──────────┐
│REGISTERED│ → │  LINKED  │ → │INSTALLING│ → │INSTALLED │ → │ VERIFIED │ → │OPERATING │
│ (등록됨) │   │ (연동됨) │   │(설치중)  │   │(설치완료)│   │ (검증됨) │   │ (운영중) │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
     │              │              │                              │              │
     │              │              │                              │              │
  단말 등록      정류장 연동    작업지시서      설치 완료       검수 승인      그룹 배정
                               생성/배정        보고                           완료
```

### 3.1 상태 정의

| 상태 | 한글명 | 설명 | 다음 가능 상태 |
|------|--------|------|---------------|
| `REGISTERED` | 등록됨 | 단말 정보만 입력됨 | `LINKED` |
| `LINKED` | 연동됨 | 정류장과 매핑 완료 | `INSTALLING` |
| `INSTALLING` | 설치중 | 작업지시서 생성, 설치 진행 중 | `INSTALLED`, `INSTALLATION_FAILED` |
| `INSTALLED` | 설치완료 | 현장 설치 완료, 검수 대기 | `VERIFIED`, `INSTALLATION_FAILED` |
| `VERIFIED` | 검증됨 | 검수 승인 완료, 정상 작동 확인 | `OPERATING` |
| `OPERATING` | 운영중 | 그룹 배정 완료, 정상 운영 | `MAINTENANCE` |
| `INSTALLATION_FAILED` | 설치실패 | 설치 실패, 재작업 필요 | `INSTALLING` |
| `MAINTENANCE` | 유지보수 | 유지보수 작업 중 | `OPERATING` |

## 4. 작업지시서 상태 전이

```
┌──────────┐   ┌──────────┐   ┌───────────┐   ┌─────────────────────┐   ┌──────────┐   ┌──────────┐
│ CREATED  │ → │ ASSIGNED │ → │IN_PROGRESS│ → │COMPLETION_SUBMITTED │ → │ APPROVED │ → │  CLOSED  │
│ (생성됨) │   │ (배정됨) │   │ (진행중)  │   │   (완료보고됨)      │   │ (승인됨) │   │ (종료됨) │
└──────────┘   └──────────┘   └───────────┘   └─────────────────────┘   └──────────┘   └──────────┘
                                    │                                        │
                                    ▼                                        ▼
                              ┌──────────┐                             ┌──────────┐
                              │ ON_HOLD  │                             │ REJECTED │
                              │ (보류됨) │                             │ (반려됨) │
                              └──────────┘                             └──────────┘
```

### 4.1 작업지시서 상태 정의

| 상태 | 한글명 | 설명 | 담당 |
|------|--------|------|------|
| `CREATED` | 생성됨 | 작업지시서 생성 완료 | 관리자 |
| `ASSIGNED` | 배정됨 | 현장 기사 배정 완료 | 관리자 |
| `IN_PROGRESS` | 진행중 | 현장 작업 진행 중 | 현장 기사 |
| `ON_HOLD` | 보류됨 | 작업 일시 중단 | 현장 기사 |
| `COMPLETION_SUBMITTED` | 완료보고됨 | 작업 완료 보고, 검수 대기 | 현장 기사 |
| `APPROVED` | 승인됨 | 검수 승인 완료 | 관리자 |
| `REJECTED` | 반려됨 | 검수 반려, 재작업 필요 | 관리자 |
| `CLOSED` | 종료됨 | 작업 최종 완료 | 시스템 |

## 7. 연관 시스템

### 7.1 CMS 연동

- **콘텐츠 배포 대상**: `OPERATING` 상태의 단말만 배포 대상에 포함
- **그룹 기반 배포**: BIS 그룹 단위로 콘텐츠 배포

### 7.2 RMS 연동

- **모니터링 대상**: `OPERATING` 상태의 단말만 모니터링
- **장애 감지**: 설치 완료 후 자동 모니터링 시작
