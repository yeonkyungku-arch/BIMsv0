# BIS 단말 모니터링 기준 및 상태 판정 시스템

**버전**: 1.0  
**작성일**: 2026-03-24  
**최종 수정**: 2026-03-24

---

## 1. 개요

BIS 단말 모니터링 시스템은 **3단계 계층적 판정 구조**를 통해 단말의 상태를 종합적으로 관리합니다:

1. **1단계**: 기기 임계치 기준 결함 판정 (하드웨어 상태)
2. **2단계**: 파생 상태 결정 (운영 환경 상태)
3. **3단계**: 복합 판정 (최종 표시 상태)

---

## 2. 1단계: 기기 임계치 기준 결함 판정

기기의 하드웨어 센서 데이터를 기반으로 결함을 판정합니다.

**판정 대상**: CPU, RAM, 내부 온도, 내부 습도, 외부 온도  
**판정 결과**: 결함 없음 / 경미한 결함 / 중대한 결함 / 치명적인 결함

### 2.1 CPU 사용량

| 결함 등급 | 임계치 범위 | 판정 코드 | 대응 시간 |
|-----------|------------|----------|----------|
| 정상 | 0% ~ 70% | normal | - |
| 경미한 결함 | 70% ~ 85% | minor | 24시간 |
| 중대한 결함 | 85% ~ 90% | major | 4시간 |
| 치명적인 결함 | 90% 이상 | critical | 1시간 |

**판정 로직**:
```
CPU 사용율 = 현재 CPU 사용량
if (CPU 사용율 >= 90) → critical (치명적)
else if (CPU 사용율 >= 85) → major (중대)
else if (CPU 사용율 >= 70) → minor (경미)
else → normal (정상)
```

### 2.2 RAM 사용량

| 결함 등급 | 임계치 범위 | 판정 코드 | 대응 시간 |
|-----------|------------|----------|----------|
| 정상 | 0% ~ 70% | normal | - |
| 경미한 결함 | 70% ~ 85% | minor | 24시간 |
| 중대한 결함 | 85% ~ 90% | major | 4시간 |
| 치명적인 결함 | 90% 이상 | critical | 1시간 |

**판정 로직**:
```
RAM 사용율 = 현재 RAM 사용량
if (RAM 사용율 >= 90) → critical (치명적)
else if (RAM 사용율 >= 85) → major (중대)
else if (RAM 사용율 >= 70) → minor (경미)
else → normal (정상)
```

### 2.3 내부 온도

| 결함 등급 | 저온 범위 | 고온 범위 | 지속 조건 | 판정 코드 |
|-----------|----------|----------|----------|----------|
| 정상 | -5°C 이상 | 40°C 이하 | - | normal |
| 경미한 결함 | -5°C 미만 | 40°C 초과 | 10분 이상 지속 / 6시간 내 | minor |
| 중대한 결함 | -15°C 미만 | 50°C 초과 | 5분 이상 반복 / 3시간 내 | major |
| 치명적인 결함 | -30°C 미만 | 60°C 초과 | 10분 이상 반복 / 1시간 내 | critical |

**판정 로직**:
```
내부온도 = 현재 측정값
지속시간 = 임계치 범위 내 지속 시간
누적시간 = 최근 N시간 내 임계치 범위 누적 시간

if ((내부온도 < -30 OR 내부온도 > 60) AND 지속시간 >= 10분) AND 누적시간 >= 60분 → critical
else if ((내부온도 < -15 OR 내부온도 > 50) AND 지속시간 >= 5분) AND 누적시간 >= 180분 → major
else if ((내부온도 < -5 OR 내부온도 > 40) AND 지속시간 >= 10분) AND 누적시간 >= 360분 → minor
else → normal
```

### 2.4 외부 온도

| 결함 등급 | 저온 범위 | 고온 범위 | 판정 코드 | 대응 조치 |
|-----------|----------|----------|----------|----------|
| 정상 | -10°C 이상 | 30°C 이하 | normal | - |
| 경미한 결함 | -10°C 미만 | 30°C 초과 | minor | 모니터링 강화 |
| 중대한 결함 | -25°C 미만 | 40°C 초과 | major | 환기/냉각 확인 |
| 치명적인 결함 | -35°C 미만 | 50°C 초과 | critical | 즉시 현장 점검 |

**판정 로직**:
```
외부온도 = 현재 측정값
if (외부온도 < -35 OR 외부온도 > 50) → critical (치명적)
else if (외부온도 < -25 OR 외부온도 > 40) → major (중대)
else if (외부온도 < -10 OR 외부온도 > 30) → minor (경미)
else → normal (정상)
```

### 2.5 내부 습도

| 결함 등급 | 저습 범위 | 고습 범위 | 지속 조건 | 판정 코드 |
|-----------|----------|----------|----------|----------|
| 정상 | 15% 이상 | 85% 이하 | - | normal |
| 경미한 결함 | 15% 미만 | 85% 초과 | 10분 이상 반복 / 6시간 내 | minor |
| 중대한 결함 | 10% 미만 | 90% 초과 | 10분 이상 반복 / 3시간 내 | major |
| 치명적인 결함 | 5% 미만 | 95% 초과 | 10분 이상 반복 / 1시간 내 | critical |

**판정 로직**:
```
내부습도 = 현재 측정값
누적시간 = 최근 N시간 내 임계치 범위 누적 시간

if ((내부습도 < 5 OR 내부습도 > 95) AND 누적시간 >= 60분) → critical
else if ((내부습도 < 10 OR 내부습도 > 90) AND 누적시간 >= 180분) → major
else if ((내부습도 < 15 OR 내부습도 > 85) AND 누적시간 >= 360분) → minor
else → normal (정상)
```

### 2.6 조도 센서

| 항목 | 내용 |
|------|------|
| 용도 | 외부 밝기 값 수집 (참조용) |
| 임계치 | 미적용 |
| 결함 판정 | 없음 |
| 표시 용도 | 디스플레이 밝기 자동 조절 참고 |

## 3. 2단계: 파생 상태 결정

기기의 운영 환경 상태를 판정합니다.

**판정 대상**: 배터리 잔량, 충전 상태, 통신 상태  
**판정 결과**: 정상 / 경고 / 장애 / 오프라인

### 3.1 배터리 잔량 (SOC - State of Charge)

| 상태 | SOC 범위 | 판정 코드 | 대응 조치 | 대응 시간 |
|------|----------|----------|----------|----------|
| 정상 | 50% 이상 | normal | - | - |
| 경고 | 20% ~ 50% | warning | 충전 스케줄 확인 | 8시간 내 |
| 장애 | 10% ~ 20% | fault | 즉시 충전 필요 | 4시간 내 |
| 긴급 | 10% 미만 | critical | 현장 출동, 배터리 교체 | 1시간 내 |

**판정 로직**:
```
SOC = 현재 배터리 잔량 %
if (SOC < 10) → critical (긴급)
else if (SOC < 20) → fault (장애)
else if (SOC < 50) → warning (경고)
else → normal (정상)
```

### 3.2 충전 상태

| 상태 | 조건 | 판정 코드 | 설명 |
|------|------|----------|------|
| 충전 중 | isCharging = true | charging | 정상 충전 진행 |
| 미충전 상태 | isCharging = false | not_charging | - |
| 충전 불량 | 미충전 시간 > 24시간 | charging_fault | 충전 회로 점검 필요 |

**판정 로직**:
```
if (isCharging = true) → charging (정상)
else if (continuousNoChargeHours > 24) → charging_fault (충전 불량)
else → not_charging (미충전)
```

**추가 조건**:
- BMS 보호 모드 활성화 → 즉시 긴급 상태로 상향
- 연속 미충전 > 24시간 + SOC < 30% → 현장 확인 필요

### 3.3 통신 상태

| 상태 | 조건 | 통신 등급 | 판정 코드 | 대응 조치 |
|------|------|----------|----------|----------|
| 정상 | connected + failCount ≤ 2 | A | normal | - |
| 주의 | connected + failCount > 2 | B | warning | 통신 상태 모니터링 |
| 불안정 | unstable OR failCount > 5 | C | warning | 신호 강도 확인, 안테나 점검 |
| 심각 | failCount > 10 | D | fault | 통신 모듈 진단 필요 |
| 두절 | disconnected | F | critical | 즉시 현장 확인 또는 원격 복구 |

**판정 로직**:
```
networkStatus = 현재 네트워크 상태
commFailCount = 최근 1시간 내 통신 실패 누적 횟수
signalStrength = 신호 강도 (dBm)

if (networkStatus = "disconnected") → critical (두절)
else if (commFailCount > 10) → fault (심각)
else if (networkStatus = "unstable" OR commFailCount > 5) → warning (불안정)
else if (commFailCount > 2) → warning (주의)
else if (networkStatus = "connected") → normal (정상)
```

**신호 강도 참고**:
- -65 dBm 이상: 우수
- -65 ~ -75 dBm: 양호
- -75 ~ -85 dBm: 보통
- -85 dBm 이하: 미흡

## 4. 3단계: 복합 판정 (최종 단말 상태)

1단계 기기 결함 판정과 2단계 파생 상태를 종합하여 **최종 5단계 상태**를 판정합니다.

**최종 상태 5단계**: 정상 / 저하 / 위험 / 오프라인 / 긴급

### 4.1 복합 판정 규칙

#### 규칙 1: 오프라인 상태

```
if (통신상태 = disconnected AND 마지막 수신 > 10분) → 오프라인
```

#### 규칙 2: 긴급 상태

```
if (기기결함 = critical AND (통신 = fault OR 배터리 < 10%)) → 긴급
else if (기기결함 = critical AND 통신 = critical) → 긴급
else if (기기결함 = critical AND (내부온도 > 60°C 또는 < -30°C)) → 긴급
```

#### 규칙 3: 위험 상태

```
if (기기결함 = critical OR major) → 위험
else if (기기결함 = minor AND 배터리 < 20%) → 위험
else if (기기결함 = minor AND 통신 = fault) → 위험
else if (통신 = critical AND 배터리 < 30%) → 위험
```

#### 규칙 4: 저하 상태

```
if (기기결함 = minor) → 저하
else if (배터리 = warning AND 통신 = warning) → 저하
else if (배터리 < 30% AND 충전불량) → 저하
else if (통신 = warning) → 저하
```

#### 규칙 5: 정상 상태

```
if (기기결함 = normal AND 배터리 >= 50% AND 통신 = normal) → 정상
```

### 4.2 복합 판정 매트릭스

| 기기 결함 | 배터리 상태 | 충전 상태 | 통신 상태 | 최종 판정 | 색상 | 긴급도 |
|---------|----------|---------|---------|---------|------|--------|
| normal | 정상 | 정상 | 정상 | **정상** | 녹색 | - |
| normal | 정상 | 정상 | warning | **저하** | 파란색 | 중간 |
| normal | warning | 정상 | 정상 | **저하** | 파란색 | 중간 |
| normal | fault | 정상 | 정상 | **위험** | 주황색 | 높음 |
| normal | critical | 정상 | 정상 | **위험** | 주황색 | 높음 |
| minor | 정상 | 정상 | 정상 | **저하** | 파란색 | 중간 |
| minor | warning | 정상 | 정상 | **저하** | 파란색 | 중간 |
| minor | fault | 정상 | 정상 | **위험** | 주황색 | 높음 |
| major | 모든 상태 | 모든 상태 | 모든 상태 | **위험** | 주황색 | 높음 |
| critical | 정상 | 정상 | 정상 | **위험** | 주황색 | 높음 |
| critical | warning | 정상 | 정상 | **위험** | 주황색 | 높음 |
| critical | fault | 모든 상태 | 모든 상태 | **긴급** | 빨간색 | 매우높음 |
| critical | critical | 모든 상태 | 모든 상태 | **긴급** | 빨간색 | 매우높음 |
| 모든 상태 | 모든 상태 | 모든 상태 | critical | **긴급** | 빨간색 | 매우높음 |
| 모든 상태 | 모든 상태 | 모든 상태 | 두절 10분+ | **오프라인** | 회색 | 높음 |

### 4.3 최종 상태별 대응

#### 정상 (NORMAL) - 녹색

```
상태: 모든 센서/항목이 정상 범위
대응: 정기 모니터링만 진행
표시: 대시보드에 초록 아이콘
```

#### 저하 (DEGRADED) - 파란색

```
상태: 단일 항목이 경미한 결함 또는 경고
대응: 
  • 상태 모니터링 강화 (10분 간격)
  • 원인 파악 및 로그 기록
  • 경보 발송 (관리자에게만)
표시: 대시보드에 파란 아이콘 + 경고 배지
```

#### 위험 (CRITICAL) - 주황색

```
상태: 중대한 결함 또는 여러 항목 이상 감지
대응:
  • 즉시 알림 발송 (관리자 + 기사)
  • 원격 진단 시도
  • 현장 출동 준비
  • 5분 간격 모니터링
표시: 대시보드에 주황 아이콘 + 경고 뱃지 + "즉시 조치" 표시
긴급 카드: "위험 단말" 카드에 표시
```

#### 긴급 (EMERGENCY) - 빨간색

```
상태: 치명적인 결함 + 운영 불가능 상태
대응:
  • 긴급 알림 발송 (모두)
  • 즉시 현장 기사 배정
  • 서비스 중단 가능성 공지
  • 1분 간격 모니터링 (자동 복구 시도)
표시: 대시보드에 빨간 아이콘 + 깜빡임 + "긴급" 표시
긴급 카드: "긴급 대응" 최상단에 표시
```

#### 오프라인 (OFFLINE) - 회색

```
상태: 10분 이상 통신 없음
대응:
  • 통신 복구 알림
  • 원격 재부팅 시도
  • 통신 장애 추정
  • 배터리 상태 확인
표시: 대시보드에 회색 아이콘 + "오프라인" 배지
긴급 카드: "원격 복구 실패" 또는 "통신 두절" 카드에 표시
```

## 5. 긴급 대응 카드 분류

대시보드 "긴급 대응" 섹션의 5가지 카드 분류:

### 5.1 즉시 조치 필요

```
조건: 긴급 상태 (red) + requiresFieldDispatch = true
내용: 치명적인 결함 + 운영 불가
대응: 즉시 현장 기사 배정
예: 내부 온도 65°C + 배터리 5%
```

### 5.2 원격 복구 실패

```
조건: 오프라인 상태 > 30분 AND 원격 복구 시도 실패
내용: 통신 복구 시도했으나 미복구
대응: 현장 확인 또는 재부팅
예: 신호 끊김 (dBm -95) + BMS 보호 모드
```

### 5.3 장시간 장애

```
조건: critical severity 알림 지속 > 2시간 AND open 상태
내용: 2시간 이상 조치 없음
대응: 현장 출동 또는 긴급 처리
예: 배터리 부족 지속 + 충전 불가
```

### 5.4 장기 미응답

```
조건: warning severity 알림 지속 > 6시간 AND open 상태
내용: 6시간 이상 경고 지속
대응: 점검 스케줄링
예: CPU 70-85% 지속 + 메모리 누수 의심
```

### 5.5 현장 출동 필요

```
조건: requiresFieldDispatch = true (수동 표시)
내용: 원격 대응 불가능
대응: 기사 배정 및 현장 방문
예: 센서 오류 의심 + 물리적 손상 가능
```

## 6. 모니터링 정책

### 6.1 데이터 수집 주기

| 항목 | 수집 주기 | 버퍼 크기 |
|------|---------|---------|
| CPU/RAM | 1분 | 60개 (1시간) |
| 내부 온도 | 5분 | 72개 (6시간) |
| 외부 온도 | 5분 | 72개 (6시간) |
| 내부 습도 | 5분 | 72개 (6시간) |
| 배터리 SOC | 1분 | 60개 (1시간) |
| 통신 상태 | 30초 | 120개 (1시간) |

### 6.2 상태 변화 트리거

```
상태 변화 시 자동 트리거:
1. 기기 결함 등급 변화 (normal → minor → major → critical)
2. 파생 상태 변화 (정상 → 경고 → 장애 → 오프라인)
3. 최종 단말 상태 변화 (정상 → 저하 → 위험 → 긴급)
4. 이력 기록 (timestamp, 변화 사유, 이전 상태)
```

## 7. 부록

### 7.1 상태 코드 요약

**기기 결함 등급**:
- `normal`: 정상 (모든 센서 정상 범위)
- `minor`: 경미한 결함 (경미 임계치 초과)
- `major`: 중대한 결함 (중대 임계치 초과)
- `critical`: 치명적인 결함 (치명 임계치 초과)

**파생 상태**:
- `normal`: 정상 (정상 범위)
- `warning`: 경고 (경고 범위)
- `fault`: 장애 (장애 범위)
- `critical`: 긴급 (긴급 범위)
- `offline`: 오프라인 (통신 두절)

**최종 단말 상태**:
- `NORMAL`: 정상 (녹색) - 모든 항목 정상
- `DEGRADED`: 저하 (파란색) - 단일 경미 결함
- `CRITICAL`: 위험 (주황색) - 중대한 결함 또는 복합 이상
- `OFFLINE`: 오프라인 (회색) - 통신 두절 10분+
- `EMERGENCY`: 긴급 (빨간색) - 치명적 결함 + 운영 불가
