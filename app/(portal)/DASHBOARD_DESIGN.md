# BIMS 메인 대시보드 설계

## 개요

BIMS 메인 대시보드는 로그인 후 첫 화면으로 표시되는 역할 기반 홈페이지입니다. 서비스 건강도를 한눈에 파악하고 빠른 대응을 가능하게 합니다.

**위치:** `/app/(portal)/page.tsx`

---

## 설계 원칙

### 1. 공유 쉘 구조 (Shared Shell Pattern)
모든 역할이 동일한 레이아웃 구조를 공유하며, 각 역할에 맞는 콘텐츠만 표시됩니다:

```
┌─ Header (PageHeader) ─────────────────────────────┐
│ 대시보드 [역할 타입]                               │
├─ Global Filter Strip ─────────────────────────────┤
│ 지역 선택 | 고객사 선택(고객사만) | 새로고침 | 내보내기 │
├─ Main Content Area ───────────────────────────────┤
│ ┌─ Summary KPI Strip (5 Cards) ─────────────────┐ │
│ │ 전체 단말 │ 정상 운영 │ 오프라인 │ 위험 │ 배터리 부족 │ │
│ └───────────────────────────────────────────────┘ │
│ ┌─ 2-Column Layout ─────────────────────────────┐ │
│ │ ┌─ Left (2/3 width) ─┬─ Right (1/3 width) ──┐ │
│ │ │ • 최근 장애       │   위치 기반 현황     │ │
│ │ │ • 진행 중인 배포 │   (지도 패널)      │ │
│ │ │ • 예정 유지보수 │                    │ │
│ │ │ • ESG 현황      │                    │ │
│ │ │ • 관리 도구      │                    │ │
│ │ └─────────────────┴────────────────────┘ │
│ └───────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────┘
```

### 2. 역할 기반 콘텐츠 필터링
코드에서 `roleType` 변수를 통해 역할 분류:

```typescript
const roleType =
  currentRole === "super_admin" || currentRole === "platform_admin"
    ? "platform"
    : currentRole === "operator"
      ? "operator"
      : currentRole === "partner"
        ? "partner"
        : "customer";
```

## 역할별 대시보드 레이아웃

### 1. 플랫폼 관리자 대시보드 (Platform Admin)
**역할:** `super_admin`, `platform_admin`

**기본 KPI:** 전체 시스템 관점
- 전체 단말: 전국 모든 단말
- 정상 운영: 평상시 운영 중인 단말
- 오프라인: 네트워크 연결 끊김
- 위험: 즉시 조치 필요
- 배터리 부족: SOC < 50%

**패널:**
- 최근 장애 (우선순위 정렬)
- 진행 중인 배포 (CMS 통계)
- ESG 현황 (전국 통계)
- 관리 도구 (콘텐츠, 배포, 계정, 역할 관리)

**필터:** 지역별 스코핑

---

### 2. 운영자 대시보드 (Operator)
**역할:** `operator`

**기본 KPI:** 운영 관점
- 전체 단말: 담당 지역 단말
- 정상 운영: 운영 중인 단말
- 오프라인: 미응답 단말
- 위험: 장애 상태
- 배터리 부족: 충전 필요

**패널:**
- 최근 장애 (즉시 대응)
- 진행 중인 배포 (배포 모니터링)
- 예정 유지보수 (금일 작업)
- ESG 현황 (지역별 통계)

**필터:** 지역별 스코핑

---

### 3. 유지보수 파트너 대시보드 (Partner)
**역할:** `partner`, `maintenance_partner`

**기본 KPI:** 현장 작업 관점
- 전체 단말: 할당된 단말
- 정상 운영: 정상 상태
- 오프라인: 수리 필요
- 위험: 긴급 수리
- 배터리 부족: 충전/교체 필요

**패널:**
- 최근 장애 (할당된 작업)
- 예정 유지보수 (오늘 일정)
- ESG 현황 (성과 기여도)

**필터:** 할당 지역

---

### 4. 고객사 관리자 대시보드 (Customer Admin)
**역할:** `customer_admin`

**기본 KPI:** 고객사 관점
- 전체 단말: 보유 단말
- 정상 운영: 정상 상태
- 오프라인: 고장
- 위험: 주의 필요
- 배터리 부족: 배터리 상태

**패널:**
- 최근 장애 (보유 단말만)
- ESG 현황 (우리 회사 통계)

**필터:** 고객사 선택, 지역 선택

### 1. Summary KPI Strip (5 Cards)
```
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   Monitor    │ │    Wifi      │ │   WifiOff    │ │ AlertTriangle│ │ BatteryWarning

│   1,247     │ │    998       │ │     34      │ │     12      │ │     47       │
```

## 전역 필터 (Global Filters)

### 1. 지역 필터 (Region Filter)
- **대상:** 모든 역할
- **기능:** mockDevices의 region으로 필터링
- **기본값:** "all" (전체 지역)
- **영향:** KPI 업데이트, 맵 표시 범위

```typescript
const regions = Array.from(new Set(mockDevices.map((d) => d.region)));
```

### 2. 고객사 필터 (Customer Filter)
- **대상:** 고객사 관리자만 표시
- **기능:** 보유 고객사 선택
- **기본값:** "all"
- **영향:** 고객사별 단말 필터링

```typescript
{roleType === "customer" && (
  <Select value={selectedCustomer} onValueChange={setSelectedCustomer}>
    {/* 고객사 선택 */}
  </Select>
)}
```

## RBAC 권한 검사

각 패널과 링크는 RBAC 권한을 확인합니다:

```typescript
{can("rms.incident.read") && (
  <Button>전체 보기</Button>
)}

{can("cms.deployment.read") && (
  <Card>진행 중인 배포</Card>
)}

{can("field_ops.work-order.read") && (
  <Card>예정 유지보수</Card>
)}

{can("analysis.lifecycle.read") && (
  <Button>상세 보기</Button>
)}
```

## 아키텍처 준수

### 1. Resolver Authority ✅
- UI에서 `displayState`, `overallState` 계산 안 함
- 백엔드에서 제공되는 상태 읽기만 함

### 2. Module Boundaries ✅
- RMS: 인시던트 관리 링크만
- CMS: 배포 현황 표시만
- Field Ops: 작업지시 현황만
- Analysis: 라이프사이클 링크만
- Admin: 관리자용 도구만

### 3. RBAC Domain Separation ✅
- 권한별 패널 표시/숨김
- rms.*, cms.*, field_ops.*, analysis.*, admin.* 분리

### 4. Reference Navigation ✅
- 모든 링크 읽기 전용
- 참조 네비게이션만 제공
- 운영 작업은 각 모듈에서만

### 5. Console IA Pattern ✅
- Header, Filter, Content 순서
- 520px 드로어 준비 (클릭 시)
- 일관된 카드 레이아웃

```
    ↓

    ↓

    ↓

    ↓

    ↓
```

## 성능 최적화

### 1. useMemo 활용
- `filteredDevices`: 필터 변경 시만 재계산
- `recentAlerts`: mockAlerts 필터링 메모이제이션
- `activeDeployments`: 배포 상태 필터링

### 2. Suspense 경계
- 지도 컴포넌트: Suspense로 감싸기
- 로딩 상태 표시: "지도 로딩 중..."

### 3. 조건부 렌더링
- 역할별 패널만 렌더링
- 권한 없는 링크 숨김
